leancheck

This is a port of Haskell's LeanCheck to Python.

WIP: This library is currently experimental. It works for simple properties, but the selection of supported types is still limited.

LeanCheck is an enumerative property-based testing library. It can be used to complement your unit tests.

The usual drill in unit testing involves making assertions about specific input-output cases of functions, such as:

assertEqual(sorted([4,2,1,3]), [1,2,3,4])

There are no arguments to the unit test.

In property-based testing (with LeanCheck) one writes more general properties that should be true for a given set of arguments. Properties in this sense are parameterized unit tests or parameterized assertions.

For example: given any list, sorting it twice is the same as sorting it once. We can encode this as a function returning a boolean value:

>>> def prop_sorted_twice(xs: list[int]) -> bool:
...     return sorted(sorted(xs)) == sorted(xs)

For whatever list we provide this function, it should return True. Now one can use LeanCheck to verify this automatically:

>>> check(prop_sorted_twice)
+++ OK, passed 360 tests: prop_sorted_twice
True

LeanCheck automatically came up with 360 unique lists to exercise the property.

When the property or function-under-test is incorrect LeanCheck may find and report a counterexample:

>>> def prop_sorted_wrong(xs: list[int]) -> bool:
...     return sorted(xs) == xs
...

>>> check(prop_sorted_wrong)
*** Failed! Falsifiable after 6 tests:
    prop_sorted_wrong([1, 0])
False

We now know that an ill input is the list [1, 0]. In this arbitrary example, the property is incorrect. In cases where the property is indeed correct, the counterexample indicates a bug.

If you have a collection of properties (prop_*) in a Python file, just call leancheck.main() and all of them will be automatically tested. In the case of a library, you can put everything under an if-expression as you would do with unittest.main() or doctest.testmod().

if __name__ == '__main__':
    leancheck.main()

150    es = [Enumerator[t] for t in types]
151    for i, args in enumerate(itertools.islice(Enumerator.product(*es), max_tests)):

154        try:

161        except PreconditionUnmatched as e:
162            continue
163        except BaseException as e:

166                print(f"*** Failed! Exception after {i+1} tests:")
167                print(f"    {red}{prop.__name__}{clear}({repr_args})")
168                print(f"    raised '{e}'")
169            return False
170    if verbose:
171        i = i+1
172        exhausted = " (exhausted)" if i < max_tests else ""
173        print(f"+++ OK, passed {i} tests{exhausted}: {green}{prop.__name__}{clear}")

196def main(max_tests=360, silent=False, verbose=False, exit_on_failure=True):
197    """
198    Tests all properties present in the current file,
199    report results and
200    exit with an error in case one of the properties fails.
201
202    This is analogous to `unittest.main()`
203    and is generally used at the end of a Python file as:
204
205        if __name__ == '__main__':
206            leancheck.main()
207    """
208    n_failures, n_properties = testmod(max_tests=max_tests, silent=silent, verbose=verbose)
209    clear, red, green, blue, yellow = _colour_escapes()
210    if not silent:
211        if not n_properties:
212            print(f"{yellow}Warning{clear}: no properties found")
213        if n_failures:
214            print(f"\n{red}*** {n_failures} of {n_properties} properties failed{clear}")
215        elif verbose:
216            print(f"{green}+++ {n_properties} properties passed{clear}")
217    if n_failures and exit_on_failure:
218        sys.exit(1)

221def testmod(max_tests=360, silent=False, verbose=False):
222    """
223    Tests all properties present in the current file
224    and report the results.
225
226    This is analogous to `doctest.testmod()`
227    and is generally used at the end of a Python file as:
228
229        if __name__ == '__main__':
230            n_failures, n_properties = leancheck.testmod()
231            if n_failures:
232                sys.exit(1)
233
234    Depending on your use-case
235    you may be better off calling `leancheck.main()` instead.
236    """
237    n_failures = 0
238    n_properties = 0
239    def lineno(m):
240        try:
241            return m[1].__code__.co_firstlineno
242        except AttributeError:
243            return -1
244    for name, member in sorted(inspect.getmembers(sys.modules["__main__"]), key=lineno):
245        if name.startswith("prop_") and callable(member):
246            n_properties += 1
247            passed = check(member, max_tests=max_tests, silent=silent, verbose=verbose)
248            if not passed:
249                n_failures += 1
250    return (n_failures, n_properties) # just like doctest.testmod()

262class Enumerator:
263    """
264    This class enumerates test values.
265
266    Enumerations are represented internally as
267    [a (potentially infinite) generator of finite lists](https://matela.com.br/thesis-rudy.pdf):
268    tiers of values of increasing size.
269    This is needed in order for the enumeration to be fair.
270
271    As a user, you can query available enumerations with
272    enumerations with "indexing":
273
274    >>> Enumerator[int]
275    Enumerator(lambda: (xs for xs in [[0], [1], [-1], [2], [-2], [3], ...]))
276
277    >>> Enumerator[bool]
278    Enumerator(lambda: (xs for xs in [[False, True]]))
279
280    >>> print(Enumerator[list[int]])
281    [[], [0], [0, 0], [1], [0, 0, 0], [0, 1], ...]
282
283    >>> print(Enumerator[tuple[bool, int]])
284    [(False, 0), (True, 0), (False, 1), (True, 1), (False, -1), (True, -1), ...]
285
286    >>> print(Enumerator[tuple[int,int,int]])
287    [(0, 0, 0), (0, 0, 1), (0, 1, 0), (1, 0, 0), (0, 0, -1), (0, 1, 1), ...]
288
289    You can use this class to build new enumerations
290    which can be registered using the `Enumerator.register()` method.
291
292    This class supports computing sums and products of enumerations:
293
294    >>> print(Enumerator[int] + Enumerator[bool])
295    [0, False, True, 1, -1, 2, ...]
296
297    Use `*` to take the product of two enumerations:
298
299    >>> print(Enumerator[int] * Enumerator[bool])
300    [(0, False), (0, True), (1, False), (1, True), (-1, False), (-1, True), ...]
301    """
302
303    tiers: typing.Callable[[], typing.Generator]
304    """
305    Generate tiers of values.

307    >>> list(Enumerator[bool].tiers())

311    def __init__(self, tiers):
312        """
313        Raw initialization of an enumerator
314        with a (potentially infinite) generator of finite lists.
315
316        >>> Enumerator(lambda: ([x] for x in itertools.count()))
317        Enumerator(lambda: (xs for xs in [[0], [1], [2], [3], [4], [5], ...]))
318
319        >>> Enumerator(lambda: (ps for ps in [[False, True]]))
320        Enumerator(lambda: (xs for xs in [[False, True]]))
321        """
322
323        self.tiers = tiers

452    @classmethod
453    def product(cls, *enumerators):
454        """
455        Computes the product of several enumerators
456        returning an enumeration of tuples.
457
458        >>> print(Enumerator.product(Enumerator[int], Enumerator[bool], Enumerator[list[int]]))
459        [(0, False, []), (0, True, []), (0, False, [0]), (0, True, [0]), (1, False, []), (1, True, []), ...]
460
461        If you have just two enumerations, you can simply use `*`:
462
463        >>> print(Enumerator[int] * Enumerator[bool])
464        [(0, False), (0, True), (1, False), (1, True), (-1, False), (-1, True), ...]
465        """
466        if len(enumerators) == 0:
467            return Enumerator(lambda: (xs for xs in [[()]]))
468        else:
469            e, *es = enumerators
470            return (e * cls.product(*es)).map(lambda t: (t[0],) + t[1])

506        Enumerator(lambda: (xs for xs in [[0], [1], [-1], [2], [-2], [3], ...]))

Enumerations are represented internally as a (potentially infinite) generator of finite lists: tiers of values of increasing size. This is needed in order for the enumeration to be fair.

As a user, you can query available enumerations with enumerations with "indexing":

>>> Enumerator[int]
Enumerator(lambda: (xs for xs in [[0], [1], [-1], [2], [-2], [3], ...]))

>>> Enumerator[bool]
Enumerator(lambda: (xs for xs in [[False, True]]))

>>> print(Enumerator[list[int]])
[[], [0], [0, 0], [1], [0, 0, 0], [0, 1], ...]

>>> print(Enumerator[tuple[bool, int]])
[(False, 0), (True, 0), (False, 1), (True, 1), (False, -1), (True, -1), ...]

>>> print(Enumerator[tuple[int,int,int]])
[(0, 0, 0), (0, 0, 1), (0, 1, 0), (1, 0, 0), (0, 0, -1), (0, 1, 1), ...]

You can use this class to build new enumerations which can be registered using the Enumerator.register() method.

This class supports computing sums and products of enumerations:

>>> print(Enumerator[int] + Enumerator[bool])
[0, False, True, 1, -1, 2, ...]

Use * to take the product of two enumerations:

>>> print(Enumerator[int] * Enumerator[bool])
[(0, False), (0, True), (1, False), (1, True), (-1, False), (-1, True), ...]
